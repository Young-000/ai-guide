## 1. 비전 & 미션

### 비전 (한 문장)

> **"AI를 처음 만나는 모든 사람이 5분 안에 자신에게 맞는 AI를 찾고, 바로 활용할 수 있는 세상"**

### 미션

AI를 전혀 모르는 일반 대중이 겪는 **"어떤 AI를 써야 할지 모르겠다"** 는 문제를 해결한다.
단순 도구 나열이 아닌, **사용자의 상황/목적/수준을 파악하여 맞춤형 가이던스를 제공**하고,
실제로 AI를 쓰기 시작할 때까지 안내하는 **온보딩 플랫폼**이다.

### v1.0 대비 변경점

| 영역 | v1.0 (도구 가이드) | v2.0 (온보딩 플랫폼) |
|------|-------------------|---------------------|
| **핵심 가치** | AI 도구 정보 제공 | 맞춤형 AI 온보딩 경험 |
| **타겟** | AI에 관심 있는 사용자 | AI를 전혀 모르는 일반 대중 |
| **수익 모델** | 없음 | 제휴 마케팅 + 프리미엄 콘텐츠 + 광고 |
| **데이터** | 정적 JSON | 정적 JSON + Supabase (사용자 데이터) |
| **개인화** | 없음 | 설문 기반 개인화 추천 + 진행 추적 |

## 3. 핵심 기능 정의 (JTBD 프레임워크)

### 3.1 스마트 온보딩 플로우

**JTBD**: *"AI에 대해 아무것도 모를 때, 몇 가지 질문에 답하면, 나에게 딱 맞는 AI 도구와 시작 방법을 알려주는 맞춤 가이드를 받고 싶다."*

#### 기능 설명

사용자가 사이트에 처음 방문하면 3~5개의 간단한 질문(상황, 목적, 경험 수준, 예산)을 통해 사용자 유형을 파악하고, 즉시 개인화된 AI 추천 결과를 보여준다.

#### 사용자 플로우

```
[랜딩 페이지] → [온보딩 시작 CTA]
  → Q1: "주로 어떤 일을 하세요?" (직업/역할)
  → Q2: "AI로 뭘 해보고 싶으세요?" (목적/상황)
  → Q3: "AI 써본 적 있으세요?" (경험 수준)
  → Q4: "예산은?" (무료/유료 선호)
  → [결과 페이지: 맞춤 추천 + 시작 가이드]
       ↓
  [가이드 실행] → [완료 체크] → [다음 단계 추천]
```

#### 범위 (MoSCoW)

**Must:**
- 3~5단계 대화형 설문 UI (카드 선택 방식, 타이핑 불필요)
- 설문 응답 기반 AI 도구 매칭 알고리즘 (규칙 기반)
- 결과 페이지: 추천 도구 + 핵심 이유 + 바로 시작하기 링크
- 설문 건너뛰기 옵션 ("바로 상황별 가이드 보기")

**Should:**
- 설문 응답 localStorage 저장 (재방문 시 복원)
- 결과 공유 기능 (URL 파라미터 기반)
- 애니메이션 전환 (부드러운 슬라이드)

**Could:**
- 대화형 챗봇 인터페이스 (타이핑 → 선택지 혼합)
- 설문 결과 기반 이메일 구독 유도

**Won't (이번 사이클):**
- AI 기반 자연어 분석 (GPT API 연동)
- 사용자 계정/로그인

### 3.2 개인화 AI 추천 엔진

**JTBD**: *"내 상황에 맞는 AI를 찾고 싶을 때, 도구 하나하나 비교하지 않고, '너한테는 이게 딱이야'라는 명확한 추천을 받고 싶다."*

#### 기능 설명

사용자의 상황(직업, 목적, 예산, 경험 수준)을 기반으로 **최적의 AI 도구 1~3개**를 추천한다. 단순 목록이 아닌, **"왜 이 도구인지"** 이유를 함께 제공한다.

#### 추천 알고리즘 (규칙 기반 v1)

```
입력: { context, task, goal, experience, budget }

매칭 로직:
1. task → situation 매핑 (taskToSituationMap)
2. experience → difficulty 필터 (초보 → easy, 중급 → medium)
3. budget → pricing.free 필터 (무료만 → free: true)
4. context → category 가중치
5. 최종 순위 = 매칭 점수 × 우선도(priority)

출력: { primaryTool, alternatives[], situation, startGuide }
```

#### 범위 (MoSCoW)

**Must:**
- 규칙 기반 매칭 알고리즘 (JSON 데이터 활용)
- 추천 결과 카드: 도구명, 추천 이유, 시작하기 버튼
- 대안 도구 2~3개 표시

**Should:**
- 추천 이유 개인화 ("김지영님은 보고서 작성이 주 업무이므로...")
- 인기도 기반 보정 (많이 클릭된 도구 가중치 상향)

**Could:**
- 협업 필터링 (비슷한 사용자가 선택한 도구)

**Won't (이번 사이클):**
- ML 기반 추천 모델
- 실시간 사용자 행동 분석

### 3.3 단계별 실습형 가이드

**JTBD**: *"AI 도구를 추천받았을 때, '시작하기' 버튼만 누르면 바로 따라할 수 있는 가이드를 보고, 5분 안에 첫 결과물을 만들고 싶다."*

#### 기능 설명

각 상황(situation)별로 **1-2-3 스텝 형태의 실습 가이드**를 제공한다. 사용자가 실제 AI 도구를 열고, 프롬프트를 복사/붙여넣기하여, 결과를 확인하는 과정까지 안내한다.

#### 가이드 구조

```
[상황 소개] → [준비물 체크] → [Step 1: 도구 접속]
  → [Step 2: 프롬프트 입력 (복사 버튼)]
  → [Step 3: 결과 확인 & 활용 팁]
  → [완료! 다음 상황 추천]
```

#### 범위 (MoSCoW)

**Must:**
- 스텝별 진행 UI (프로그레스 바 + 이전/다음)
- 프롬프트 원클릭 복사 기능
- 각 상황별 최소 2개 프롬프트 템플릿
- 난이도 표시 (쉬움/보통/어려움)
- 예상 소요 시간 표시

**Should:**
- 스텝 완료 체크 기능 (localStorage)
- "이 가이드가 도움이 됐나요?" 피드백 버튼
- 관련 상황 추천 (이 가이드를 완료한 사람은 이것도 봤어요)

**Could:**
- 스크린샷/GIF로 시각적 안내
- 프롬프트 커스터마이즈 인풋 (변수 부분만 입력)

**Won't (이번 사이클):**
- 영상 가이드
- AI API 직접 연동 (인앱 실행)

### 3.4 활용 사례 라이브러리

**JTBD**: *"AI로 뭘 할 수 있는지 감이 안 올 때, 나와 비슷한 사람들이 AI를 어떻게 활용하는지 실제 사례를 보고, 영감을 얻고 싶다."*

#### 기능 설명

직업별/상황별로 **"이 사람은 이렇게 AI를 활용합니다"** 형태의 사례를 정리한다. 각 사례는 Before/After, 절약 시간, 사용 도구를 포함한다.

#### 사례 구조

```json
{
  "persona": "마케팅 팀원",
  "situation": "주간 보고서 작성",
  "tool": "Claude",
  "before": "매주 3시간 소요, 데이터 수집~정리~작성",
  "after": "30분으로 단축, AI가 초안 작성 → 수정만",
  "timeSaved": "2시간 30분/주",
  "prompt": "...",
  "tags": ["업무", "보고서", "시간절약"]
}
```

#### 범위 (MoSCoW)

**Must:**
- 최소 15개 활용 사례 (직업별 3개 이상)
- 카테고리/태그 필터링
- Before/After 비교 시각화

**Should:**
- 사례별 "나도 해보기" CTA → 관련 가이드로 연결
- 인기순/최신순 정렬

**Could:**
- 사용자 제출 사례 (UGC)
- 시간 절약 계산기 ("이만큼 절약할 수 있어요")

**Won't (이번 사이클):**
- 사용자 로그인/커뮤니티

### 3.5 진행 상황 추적 (레벨 시스템)

**JTBD**: *"AI 활용을 시작했을 때, 내가 어디까지 배웠는지 확인하고, 다음에 뭘 해볼지 가이드를 받아서, 꾸준히 AI 활용 능력을 키우고 싶다."*

#### 기능 설명

사용자가 가이드를 완료할 때마다 경험치(XP)를 부여하고, 레벨업 시스템으로 성취감을 제공한다. 다음 추천 상황을 자동으로 제안하여 리텐션을 높인다.

#### 레벨 시스템

```
Lv.1 AI 새싹 (0-49 XP)     → 기본 가이드 1~2개 완료
Lv.2 AI 탐험가 (50-149 XP)  → 3개 카테고리 경험
Lv.3 AI 활용가 (150-299 XP) → 고급 가이드 완료
Lv.4 AI 달인 (300+ XP)      → 10개 이상 가이드 완료
```

#### 범위 (MoSCoW)

**Must:**
- localStorage 기반 진행 상황 저장
- 완료한 가이드 목록 + XP 표시
- 레벨 표시 (프로필 뱃지)
- "다음에 해보세요" 추천

**Should:**
- 레벨업 시 축하 애니메이션
- 카테고리별 진행률 시각화

**Could:**
- Supabase 연동으로 크로스 디바이스 동기화
- 주간 리포트 ("이번 주 AI 활용 요약")

**Won't (이번 사이클):**
- 리더보드/랭킹
- 소셜 공유 기능

### 6.2 Supabase 결정: Phase 3에서 도입

**Phase 1~2에서 Supabase 불필요한 이유:**

1. 사용자 계정/로그인이 MVP에 없음
2. 모든 콘텐츠는 정적 JSON으로 충분
3. 진행 추적은 localStorage로 가능
4. 인프라 복잡도 최소화 우선

**Phase 3에서 Supabase 도입 시 스키마 (예비 설계):**

```sql
-- Supabase Project 2 (비게임 앱) 사용
-- 스키마: ai_guide

CREATE SCHEMA IF NOT EXISTS ai_guide;

-- 사용자 프로필 (Supabase Auth 연동)
CREATE TABLE ai_guide.user_profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  display_name TEXT,
  persona_type TEXT, -- 'office_worker' | 'student' | 'creator' | 'business_owner'
  experience_level TEXT, -- 'never' | 'basic' | 'some' | 'expert'
  onboarding_answers JSONB,
  total_xp INTEGER DEFAULT 0,
  current_level INTEGER DEFAULT 1,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 가이드 완료 기록
CREATE TABLE ai_guide.guide_completions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES ai_guide.user_profiles(id),
  situation_slug TEXT NOT NULL,
  completed_at TIMESTAMPTZ DEFAULT NOW(),
  xp_earned INTEGER DEFAULT 0,
  feedback_rating INTEGER, -- 1-5
  UNIQUE(user_id, situation_slug)
);

-- 제휴 링크 클릭 추적
CREATE TABLE ai_guide.affiliate_clicks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tool_slug TEXT NOT NULL,
  source_page TEXT NOT NULL, -- 클릭 발생 페이지
  session_id TEXT,
  clicked_at TIMESTAMPTZ DEFAULT NOW()
);

-- 프리미엄 콘텐츠 구매
CREATE TABLE ai_guide.purchases (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES ai_guide.user_profiles(id),
  content_type TEXT NOT NULL, -- 'prompt_pack' | 'course' | 'report'
  content_id TEXT NOT NULL,
  amount INTEGER NOT NULL, -- 원 단위
  purchased_at TIMESTAMPTZ DEFAULT NOW()
);
```

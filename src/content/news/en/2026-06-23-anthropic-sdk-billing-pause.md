**Summary**: Anthropic reversed a planned billing change for its Claude Agent SDK on June 15 — the same day it was supposed to go live — leaving developer subscription limits intact for now.

### Key facts
- The original plan (announced May 13): a **separate monthly credit pool** for Agent SDK, `claude -p`, and GitHub Actions usage — **$20 for Pro, $100 for Max 5x, $200 for Max 20x** — billed at standard API rates
- The change was scheduled for June 15; Anthropic quietly pulled it on that same day with no prior warning
- **Current state**: Agent SDK usage continues to draw from normal subscription limits — no separate credit pool in effect
- Analysts point to three contributing factors: developer community backlash, OpenAI reportedly weighing steep API price cuts, and Anthropic's IPO preparation (developer goodwill matters more during a public offering run-up)

### Why it matters
Monetizing agentic, automated workloads is an industry-wide challenge for subscription-based AI platforms. Anthropic's last-minute reversal illustrates the tension between growing revenue and preserving the developer base that drives platform adoption. Whether and when this policy returns will be closely watched, especially as the company heads toward its IPO.

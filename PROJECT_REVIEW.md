## 1. 프로젝트 요약

### 핵심 가치 제안

> **"AI를 처음 접하는 사람이 '무엇을 해야 하는지' 말하면, 바로 쓸 수 있는 도구와 방법을 알려준다"**

| 항목 | 내용 |
|------|------|
| **타겟 사용자** | AI를 처음 접하는 직장인, 학생, 일반인 |
| **핵심 문제** | "AI가 좋다는데... 뭘 어떻게 쓰지?" |
| **해결 방식** | 상황 → 도구 + 프롬프트 + 단계별 가이드 |
| **차별점** | 도구 나열 X, 문제 해결 중심 O |

### 피벗 히스토리

| 버전 | 날짜 | 접근 방식 | 한계 |
|------|------|----------|------|
| v1.0 | 2025-01 | 도구 중심 (10개 AI 도구 소개) | "그래서 뭘 써야 해?" |
| **v2.0** | 2025-01~ | **상황 기반** (문제→해결책) | - |

### 현재 상태

```
✅ MVP 완성 (프론트엔드 + 정적 데이터)
✅ Vercel 배포 완료
✅ 핵심 기능 구현 (검색, 가이드, 설문, 레벨업)
✅ 상황 데이터 21개로 확장 완료
✅ 검색 동의어 개선 (한국어 음차 표현 추가)
✅ 접근성(A11y) 개선 완료 (aria-label, focus 관리)
✅ 토이 프로젝트 가이드 페이지 추가
✅ 빌드 검증 완료 (33개 정적 페이지)
✅ SEO/메타태그 최적화 완료 (OpenGraph, Twitter Card)
✅ 소셜 공유 기능 추가 (Web Share API + 클립보드 폴백)
✅ 사용자 피드백 위젯 추가 (이모지 평가 + 의견 수집)
❌ 백엔드/DB 미구현 (Phase 2)
```

## 2. 개발 관점 리뷰

### 2.1 기술 스택 평가

| 분류 | 기술 | 적합도 | 평가 |
|------|------|:------:|------|
| **Framework** | Next.js 14 | ⭐⭐⭐⭐⭐ | App Router, SSG 활용으로 빠른 초기 로딩 |
| **Language** | TypeScript | ⭐⭐⭐⭐⭐ | 타입 안전성으로 유지보수성 확보 |
| **Styling** | Tailwind CSS | ⭐⭐⭐⭐⭐ | 빠른 UI 개발, 일관된 디자인 |
| **Data** | JSON (정적) | ⭐⭐⭐☆☆ | MVP에 적합, 확장성 제한 |
| **Storage** | LocalStorage | ⭐⭐⭐☆☆ | 간단한 진도 저장, 동기화 불가 |

### 2.2 아키텍처 분석

```
src/
├── app/                 # 페이지 라우팅 (Next.js App Router)
│   ├── page.tsx         # 메인 (상황 검색) ⭐ 핵심
│   ├── situations/      # 상황 목록 + 상세
│   ├── tools/           # 도구 목록 + 상세
│   ├── quiz/            # AI 추천 테스트
│   └── ...
├── components/          # 재사용 UI 컴포넌트
│   ├── GuidePanel.tsx   # 핵심: 상황별 가이드 ⭐
│   ├── SurveyWizard.tsx # 핵심: 설문 기반 추천 ⭐
│   ├── OnboardingModal.tsx
│   ├── LevelBadge.tsx
│   └── ...
├── lib/                 # 비즈니스 로직
│   ├── search.ts        # 검색 알고리즘
│   ├── surveyLogic.ts   # 설문 추천 로직
│   ├── levelSystem.ts   # 레벨/XP 시스템
│   └── synonyms.ts      # 동의어 확장
└── data/                # 정적 데이터 (JSON)
    ├── situations.json  # 상황 데이터 ⭐
    ├── tools.json       # 도구 데이터
    └── survey.json      # 설문 데이터
```

**장점:**
- 관심사 분리 명확 (UI / 로직 / 데이터)
- 컴포넌트 재사용성 높음
- 정적 생성(SSG)으로 빠른 로딩

**개선점:**
- `data/` JSON 파일이 커지면 빌드 시간 증가 예상
- API 연동 시 구조 변경 필요 (Phase 2)

### 2.3 코드 품질

| 항목 | 상태 | 비고 |
|------|:----:|------|
| TypeScript 커버리지 | ✅ 100% | 모든 타입 정의됨 |
| ESLint | ✅ | 규칙 준수 |
| 컴포넌트 테스트 | ⚠️ 부족 | Unit 테스트 미흡 |
| E2E 테스트 | ✅ | Playwright 설정됨 |
| setTimeout cleanup | ✅ | 메모리 누수 방지 처리 완료 |

### 2.4 성능

| 지표 | 현재 | 목표 | 평가 |
|------|------|------|------|

**정적 생성(SSG) 덕분에 초기 로딩이 빠름** - AI 초심자가 기다리지 않고 바로 사용 가능

## 3. 기능 관점 리뷰

### 3.1 핵심 사용자 여정

```
[첫 방문] → [온보딩] → [상황 검색/설문] → [가이드 확인] → [실제 사용] → [레벨업]
```

#### 3.1.1 온보딩 (신규 사용자 첫 경험)

| 평가 항목 | 점수 | 설명 |
|----------|:----:|------|
| **직관성** | ⭐⭐⭐⭐☆ | 4단계 슬라이드로 서비스 설명 |
| **부담감** | ⭐⭐⭐⭐⭐ | 스킵 가능, 강제성 없음 |
| **학습 효과** | ⭐⭐⭐☆☆ | 기능 설명만, 실습 연동 없음 |

**AI 초심자 관점:**
- ✅ "뭐하는 서비스인지" 빠르게 파악 가능
- ⚠️ 첫 번째 상황을 직접 체험하게 유도하면 더 효과적

#### 3.1.2 상황 검색

| 평가 항목 | 점수 | 설명 |
|----------|:----:|------|
| **자연어 이해** | ⭐⭐⭐⭐☆ | "PDF 요약해줘" 같은 구어체 인식 |
| **동의어 처리** | ⭐⭐⭐⭐☆ | synonyms.ts로 유사어 확장 |
| **검색 정확도** | ⭐⭐⭐☆☆ | 정확 매칭 의존, 의미 검색 없음 |
| **검색 결과 표시** | ⭐⭐⭐⭐⭐ | 매칭된 키워드 하이라이트 |

**AI 초심자 관점:**
- ✅ "뭐라고 검색해야 하지?" 고민 줄여줌
- ✅ 한국어 음차 표현 지원 ("챗지피티" → ChatGPT, "클로드" → Claude 등)
- 💡 개선안: fuzzy matching 또는 LLM 기반 의도 분석

#### 3.1.3 설문 기반 추천 ("뭘 해야 할지 모르겠어요")

| 평가 항목 | 점수 | 설명 |
|----------|:----:|------|
| **질문 설계** | ⭐⭐⭐⭐⭐ | 상황→작업→긴급도→품질→경험→예산 |
| **동적 옵션** | ⭐⭐⭐⭐☆ | 이전 답변에 따라 선택지 변화 |
| **추천 정확도** | ⭐⭐⭐⭐☆ | taskToSituationMap 활용 |
| **결과 설명** | ⭐⭐⭐⭐⭐ | "왜 이걸 추천하는지" 설명 제공 |

**AI 초심자 관점:**
- ✅ **가장 중요한 기능** - 검색어를 모르는 사용자를 위한 길잡이
- ✅ "무료만", "초보자용" 필터로 진입 장벽 낮춤
- ⚠️ 6개 질문이 약간 길게 느껴질 수 있음

#### 3.1.4 상황별 가이드 (GuidePanel)

| 평가 항목 | 점수 | 설명 |
|----------|:----:|------|
| **단계별 안내** | ⭐⭐⭐⭐⭐ | 1-2-3-4 순서로 명확한 가이드 |
| **프롬프트 제공** | ⭐⭐⭐⭐⭐ | 복사 버튼으로 바로 사용 가능 |
| **도구 추천** | ⭐⭐⭐⭐☆ | 주 추천 + 대안 도구 제시 |
| **난이도 표시** | ⭐⭐⭐⭐⭐ | 쉬움/보통/어려움 시각화 |
| **예상 시간** | ⭐⭐⭐⭐⭐ | "5-10분" 같은 구체적 시간 |

**AI 초심자 관점:**
- ✅ **핵심 가치 전달** - "이거 복사해서 붙여넣기만 하면 돼"
- ✅ 난이도와 시간으로 "내가 할 수 있을까?" 불안 해소
- ⚠️ 실제 AI 도구 화면 스크린샷 없음 (시각적 안내 부족)

#### 3.1.5 레벨업 시스템 (게이미피케이션)

| 레벨 | 타이틀 | XP 범위 | 심리적 효과 |
|:----:|--------|---------|------------|
| 1 | AI 새싹 🌱 | 0-99 | "시작했어!" |
| 2 | AI 탐험가 🔍 | 100-299 | "좀 알아가고 있어" |
| 3 | AI 실험가 🧪 | 300-599 | "직접 써봤어" |
| 4 | AI 숙련자 ⚡ | 600-999 | "이제 좀 익숙해" |
| 5 | AI 마스터 🏆 | 1000+ | "내가 알려줄 수 있어" |

**XP 보상 체계:**
| 행동 | XP |
|------|---:|
| 첫 방문 (온보딩 완료) | +10 |
| 스텝 1개 완료 | +10 |
| 상황 전체 완료 보너스 | +30 |

**AI 초심자 관점:**
- ✅ 학습 동기 부여 (XP 획득의 작은 성취감)
- ✅ "나도 성장하고 있다"는 시각적 피드백
- ⚠️ 현재 보상이 뱃지뿐 → 추가 인센티브 고려 필요

### 3.2 콘텐츠 품질

| 항목 | 현재 수량 | 품질 평가 | 개선점 |
|------|:--------:|:--------:|--------|
| **상황** | 21개 ✅ | ⭐⭐⭐⭐⭐ | 번역, 이력서, 엑셀, SNS 등 실무 상황 추가 |
| **도구** | 21개 ✅ | ⭐⭐⭐⭐⭐ | 주요 AI 도구 모두 포함 |

| **토이 프로젝트** | 6개 ✅ | ⭐⭐⭐⭐⭐ | AI 초심자용 실습 가이드 추가 |

## 4. 비즈니스 관점 리뷰

### 4.1 시장 분석

#### 타겟 세그먼트

| 세그먼트 | 특성 | 이 서비스의 가치 |
|----------|------|----------------|
| **직장인 (30-50대)** | AI 써보고 싶지만 막막함 | "회사에서 바로 쓸 수 있는 가이드" |
| **대학생** | 과제/논문에 AI 활용 희망 | "레포트 쓸 때 이렇게 쓰면 돼" |
| **프리랜서** | 생산성 도구 관심 높음 | "작업별 최적 도구 추천" |
| **시니어** | 디지털 리터러시 향상 의지 | "쉬운 것부터 차근차근" |

#### 경쟁 서비스 비교

| 서비스 | 접근 방식 | 한계점 | AI Guide 차별점 |
|--------|----------|--------|----------------|
| **공식 문서** (Claude, GPT) | 기능 나열 | 뭘 해야 할지 모르면 막막 | 상황 기반 접근 |
| **유튜브 튜토리얼** | 영상 시청 | 시간 소요, 검색 어려움 | 텍스트로 빠르게 확인 |
| **블로그/뉴스레터** | 팁 공유 | 파편화, 체계 없음 | 구조화된 가이드 |
| **AI 도구 비교 사이트** | 스펙 나열 | "그래서 뭘 써야 해?" | 상황→도구 추천 |

### 4.2 핵심 가치 제안 (Value Proposition)

```
┌─────────────────────────────────────────────────────────────┐
│  [문제] AI가 좋다는데, 뭘 어떻게 시작해야 할지 모르겠어요     │
├─────────────────────────────────────────────────────────────┤
│  [기존 해결책의 한계]                                        │
│  - 공식 문서: 너무 어렵고 기능 중심                          │
│  - 유튜브: 시간 많이 들고 원하는 내용 찾기 어려움             │
│  - 블로그: 파편화되어 있고 신뢰도 불확실                      │
├─────────────────────────────────────────────────────────────┤
│  [AI Guide의 해결책]                                         │
│  ✅ "뭐 하고 싶어요?" → 즉시 실행 가능한 가이드 제공          │
│  ✅ 프롬프트 복사 → 붙여넣기만 하면 끝                        │
│  ✅ 단계별 안내 → 처음 해도 따라갈 수 있음                    │
└─────────────────────────────────────────────────────────────┘
```
